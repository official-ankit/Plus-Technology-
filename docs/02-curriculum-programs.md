## Track 1: C++ Programming

### Learning Objectives
Upon completion, students will be able to:
- Write, compile, and debug C++ programs
- Apply OOP principles (encapsulation, inheritance, polymorphism)
- Use STL containers and algorithms effectively
- Implement data structures from scratch
- Solve competitive programming problems

### Module Breakdown

| Module | Topics | Hours | Deliverables |
|--------|--------|-------|--------------|
| **M1: Foundations** | Syntax, variables, operators, I/O, control flow | 15 | 5 quizzes, 3 coding labs |
| **M2: Functions & Arrays** | Functions, recursion, arrays, strings | 15 | 5 quizzes, 4 labs, mini-project |
| **M3: Pointers & Memory** | Pointers, references, dynamic memory, smart pointers | 20 | 5 quizzes, 5 labs |
| **M4: OOP Basics** | Classes, objects, constructors, access specifiers | 15 | 4 quizzes, 4 labs |
| **M5: OOP Advanced** | Inheritance, polymorphism, virtual functions, abstract classes | 15 | 4 quizzes, 4 labs, mini-project |
| **M6: STL & Templates** | Vectors, maps, sets, algorithms, templates | 15 | 5 quizzes, 5 labs |
| **M7: File I/O & Exceptions** | File handling, exception handling, debugging | 10 | 3 quizzes, 3 labs |
| **M8: Capstone** | Real-world project implementation | 15 | Project + code review |

### Sample Lab: Linked List Implementation
```
Objective: Implement a singly linked list with insert, delete, search, reverse operations.
Expected Output: Passing all 15 unit tests, O(n) complexity analysis.
Time: 90 minutes
```

### Sample Capstone Project Ideas
1. **Student Management System** — CRUD operations, file persistence, search/sort
2. **Mini Compiler** — Lexer + parser for a simple language subset
3. **Game Engine Basics** — 2D game loop with OOP design patterns

## Hands-On Components

### Labs
- **Format**: Guided coding exercises with starter code
- **Auto-grading**: Unit tests run in sandbox
- **Hints**: Progressive hints unlocked after 2 failed attempts
- **Time**: 30–90 minutes per lab

### Mini-Projects
- **Format**: Open-ended problem with requirements spec
- **Submission**: GitHub repo or in-platform upload
- **Review**: Automated + peer/mentor code review
- **Examples**:
  - Calculator app (C++)
  - Tic-tac-toe (Python)
  - Todo REST API (Node.js)

### Capstone Projects
- **Requirements**: Detailed brief with acceptance criteria
- **Duration**: 2–4 weeks
- **Deliverables**: Code, README, demo video (optional)
- **Grading**: Rubric-based (functionality 40%, code quality 30%, documentation 20%, creativity 10%)

## Assessment Methods

| Type | Weight | Format | Frequency |
|------|--------|--------|-----------|
| **Quizzes** | 15% | MCQ, auto-graded | End of each topic |
| **Labs** | 25% | Coding, auto-graded | 2–4 per module |
| **Assignments** | 20% | Coding + written | 1–2 per module |
| **Peer Code Reviews** | 10% | Review 2 peers per project | Per mini-project |
| **Proctored Exam** | 30% | MCQ + coding | End of track |

### Grading Rubrics

#### Code Quality Rubric (Labs/Projects)
| Criteria | Excellent (4) | Good (3) | Satisfactory (2) | Needs Work (1) |
|----------|--------------|----------|------------------|----------------|
| **Correctness** | All tests pass | 90%+ tests pass | 70%+ tests pass | <70% pass |
| **Readability** | Clear naming, comments | Mostly readable | Some issues | Hard to follow |
| **Efficiency** | Optimal complexity | Near-optimal | Acceptable | Inefficient |
| **Edge Cases** | All handled | Most handled | Some handled | None handled |

## Delivery Modes

### Self-Paced
- Access all content immediately
- Suggested weekly schedule provided
- Async mentor support (24–48h response)

### Live/Hybrid
- Scheduled live sessions 2x/week
- Recorded replays available
- Real-time Q&A, breakout rooms

### Cohort-Based
- Fixed start/end dates
- Peer learning groups
- Weekly milestones, deadlines

## Accessibility & Accommodations

### Content Accessibility
- All videos captioned (auto + human-reviewed)
- Transcripts for all audio content
- Alt text for all images/diagrams
- Keyboard-navigable code editor
- Screen reader compatible UI

### Accommodations
- Extended time for exams (1.5x – 2x)
- Flexible deadlines (case-by-case)
- Alternative formats (audio instructions, larger text)
- Sign language interpreter requests (live sessions)

### Inclusive Design Principles
- Gender-neutral language
- Diverse representation in examples
- Time-zone friendly scheduling (recordings available)
- Low-bandwidth mode (text-first, optional video)
